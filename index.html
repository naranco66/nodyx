Pourquoi Nexus existe

Vos communautés méritent
mieux que ça.

Discord, Matrix, TeamSpeak ont tous des compromis inacceptables. Nexus n'en a aucun.

Discord
Vos données, leur produit
  • Données vendues à des annonceurs
  • Serveurs fermés, non indexés par Google
  • Code propriétaire, boîte noire totale
  • Peut fermer du jour au lendemain
  • Compte requis pour lire quoi que ce soit
Matrix / Element
Complexe et fragmenté
  • Pas de forum structuré, pas de SEO
  • UX difficile pour les non-techs
  • Pas de vocal P2P natif intégré
  • Installation lourde et technique
  • Pas de calendrier, pas de polls
Nexus
Tout le reste, sans les compromis
  • Forum indexé par Google, Bing, Brave
  • Chat + vocal + DMs + calendrier intégrés
  • Auto-hébergé, vos données restent chez vous
  • P2P distribué, aucun serveur central
  • Open source AGPL — lisible et forkable

Vocal & Temps réel

P2P natif.
Pas de serveur vocal central.

Le son passe directement d'un navigateur à l'autre via WebRTC. Nexus n'écoute rien.

Architecture WebRTC
01
Mesh P2P direct
Chaque participant est connecté à tous les autres. Aucun audio ne transite par le serveur Nexus.
WebRTC · ICE · DTLS
02
Signalisation Socket.IO
Le serveur sert uniquement à l'échange des offres SDP et candidats ICE — pas d'audio.
SDP · ICE Candidates
03
nexus-turn en fallback
Derrière VPN, CGNAT ou réseau d'entreprise — nexus-turn (Rust) garantit la connexion.
STUN/TURN · Rust
04
DataChannels en parallèle
Saisie instantanée, réactions optimistes, transfert de fichiers — sur le même mesh P2P.
RTCDataChannel · P2P

Installation

Votre instance,
en 5 minutes.

Node.js 20+, PostgreSQL 16, Redis 7. Ou juste Docker.

Recommandé
Docker
La méthode la plus simple. Toutes les dépendances incluses dans les conteneurs.
# Cloner le dépôt git clone https://github.com/Pokled/Nexus cd Nexus/nexus-core # Configurer votre communauté cp .env.example .env # Démarrer docker-compose up -d # API disponible sur :3000
Manuel (Linux/Mac)
Contrôle total sur chaque composant. Recommandé pour les serveurs de production.
# Prérequis : Node 20+, PG 16, Redis 7 cd nexus-core && npm install cp .env.example .env # Éditer .env puis : npm run dev # développement npm run build # production npm start # démarrer
Windows (PowerShell)
Script d'installation automatique — Node.js, PostgreSQL, Redis, service Windows.
# Ouvrir PowerShell en Administrateur .\scripts\Install-Nexus.ps1 # Ou chemin personnalisé : .\scripts\Install-Nexus.ps1 \ -NexusPath "D:\Apps\Nexus" # Installe Chocolatey, Node, PG, # Redis, NSSM (service auto-start)
Reverse proxy recommandé
Caddy — SSL automatique
Caddy gère vos certificats SSL via Let's Encrypt automatiquement. Zéro configuration manuelle. Un fichier Caddyfile suffit.
# Caddyfile minimal votre-domaine.com { reverse_proxy /api/* localhost:3000 reverse_proxy /socket.io/* localhost:3000 reverse_proxy * localhost:4173 }
